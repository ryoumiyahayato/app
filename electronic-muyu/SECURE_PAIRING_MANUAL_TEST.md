# 安全配对双真机验收清单

## 准备

1. 先部署经过 `npm run check` 和 integration 验证的 Worker，保持 `ALLOW_LEGACY=false`。
2. 确认 `https://<自定义域名>/health` 返回协议版本 1，且 workers.dev/preview URL 不可用。
3. 使用同一 HTTPS base URL 构建两个 Debug/候选 Release APK；不要在 QR 中放域名。
4. 两台 Android 8.0+ 设备分别全新安装，记录系统版本、ROM、网络和省电设置。

## 配对

- [ ] 未配对设置页没有服务器、room、token 输入框。
- [ ] 只有进入扫码页时请求相机权限；拒绝权限可安全返回。
- [ ] A 创建二维码，显示 120 秒以内倒计时；取消后 B 扫描/join 失败。
- [ ] 重新生成后旧 QR 失败，新 QR 可用。
- [ ] B 扫码不打开 URL、不写剪贴板，显示“正在建立安全连接”。
- [ ] 两机显示完全一致、零填充的 6 位 SAS。
- [ ] 一端点“不一致”，两端不能形成长期配对。
- [ ] 仅一端确认时不能连接；双方确认后才自动连接。
- [ ] 已配对页只显示非敏感对端名称/短状态。

## 消息与网络

- [ ] A/B 前台各连续点击 60 次，合法突发不被误限流，计数、声音和震动次数正确。
- [ ] 本机敲击使用木鱼音效和短震动；收到对方提醒使用独立提示音和通知震动模式，两者不会串用。
- [ ] 对方上线、离线和重新连接时，主界面与常驻通知状态及时更新；离线期间的提醒不补发。
- [ ] 两机分别处于前台、后台、锁屏时互发；解密失败不产生任何提醒。
- [ ] 点击系统通知能回到 App，Activity 重建窗口不重复播放也不丢失 10 秒内事件。
- [ ] Wi-Fi ↔ 蜂窝切换后自动重连并重新认证。
- [ ] 飞行模式 2 分钟后恢复，退避没有形成高频循环。
- [ ] 连接处于“正在重连”时快速点击 10 次；10 秒内恢复后按原顺序全部送达，不丢前几次。
- [ ] 断网超过 10 秒时，暂存提醒被丢弃并明确显示“部分提醒未发送”。
- [ ] 模拟 4408/HTTP 429 后至少冷却 15 秒并自动恢复，不需要用户重新点连接。
- [ ] 同一设备模拟第二连接被 4409 拒绝；未认证连接 5 秒后 4410。
- [ ] 重放相同 envelope/counter 被 4414 拒绝且对端不增加计数。

## 撤销与迁移

- [ ] “解除配对”弹二次确认；取消不改变状态。
- [ ] 确认解除后两端在线 socket 关闭，旧 token 无法重新认证。
- [ ] 网络断开时解除失败，本机明确提示且保留可再次撤销的凭据。
- [ ] 从 v0.6 带旧 URL/room 升级：不自动连接，显示迁移提示；新配对完成后旧字段删除。

## 长时间与发布

- [ ] Android 15/16 各连续运行至少 24 小时，记录 `remoteMessaging` FGS、Doze、厂商杀后台情况。
- [ ] Release 使用生产 BuildConfig HTTPS origin，R8、签名、安装、升级安装通过。
- [ ] Play Console 已人工声明 `remoteMessaging` 的真实用途。
- [ ] 检查 Android/Worker 应用日志没有 QR、secret/token、完整 ID、私钥、密钥或明文 tap。
- [ ] Cloudflare 生产环境验证未认证 socket 的 4410 alarm 关闭帧可被真实 Android 客户端收到。

真机、公网、Play 审核结果必须由执行者单独记录；代码构建通过不能替代本清单。
